CPT（地理信息基座模型）工作总结

  一句话介绍

  CPT项目旨在构建高德地图地理信息垂域基座大模型，基于Qwen3-1.7B，通过大规模地理 
  垂域数据（8200万POI +
  157万道路）的持续预训练（Mid-training），使模型具备对POI、道路网络等地理实体的
  深度理解能力。项目涵盖从垂域数据建设、编码方案设计、训练框架搭建到评测体系构建
  的完整链路，使用Open-Megatron-LLaMA框架在64卡集群上完成10B tokens的训练。     

  ---
  详细工作步骤

  1. 技术调研

  - 垂域模型训练方案调研：系统梳理了120+篇垂域大模型论文，涵盖医疗（HuatuoGPT） 
  、金融（FinGPT）、法律（ChatLaw）等领域，总结出垂域模型的通用训练范式：CPT（持
  续预训练）→ SFT（指令微调）→ RLHF。关键结论包括：垂域数据与通用数据的配比通常 
  在1:5~1:20之间；学习率通常为基座预训练的1/10~1/5；数据质量比数量更重要。      
  - 地图地理垂域大模型调研：分析了百度MapGPT、腾讯地图大模型、CityGPT、UrbanGPT 
  等竞品方案，确认了地理信息垂域模型的技术可行性和差异化方向。
  - Mid-training技术调研：研究了DeepSeek、OLMo、GLM、Qwen等主流模型的mid-trainin
  g策略，重点关注数据配比、学习率调度（WSD/cosine）、micro-annealing验证方法等。

  2. 数据建设

  2.1 POI数据建设

  - 数据规模：全量8200万条POI，覆盖21个大类（餐饮服务、购物服务、交通设施等）。 
  - 位置编码方案：对比评估了Google S2、Geohash、Uber
  H3三种地理编码方案，最终选择Google S2多尺度编码（L4国家级/L10城市级/L20街道级 
  ），因其具有层级一致性、面积均匀性和空间保序性优势。
  - 文本合成：设计了system prompt，利用大模型将POI结构化信息（名称、地址、类目、
  坐标、评分等）合成为自然语言描述，并设计了验证prompt确保生成质量。
  - 类目统计与筛选：对全量数据按类目、PV进行统计分析，筛选出高质量子集用于训练。

  2.2 道路数据建设

  - 数据规模：157万条道路link数据，来源于gd_cdm.dim_gd_link_1st_ver表。
  - Link拼接：将零散的link按road_id拼接为完整道路，保持拓扑顺序。
  - 四种文本格式转换（通过road_info_to_pretrain.py工具）：
    - 自然语言格式：将道路信息转化为流畅的描述文本，推荐用于预训练
    - 结构化格式：保留字段结构的文本化表达
    - QA问答对格式：围绕道路属性生成问答对，推荐用于指令微调
    - 知识图谱三元组格式：提取道路实体关系三元组
  - 字段分类：将50+个道路字段分为身份命名、地理位置、几何属性、拓扑关系、车道信 
  息、功能标识六大类。

  2.3 通用数据建设

  - 收集整理通用预训练数据集，进行清洗、去重处理。
  - 最终垂域与通用数据配比约为1:14（domain:general）。

  3. 训练框架搭建

  - 框架选型：采用Open-Megatron-LLaMA（基于Megatron-LM），支持张量并行（TP）、流
  水线并行（PP）、数据并行（DP）。
  - Qwen3适配：解决了Qwen3-1.7B在Megatron中的适配问题，包括：
    - 词表大小设置（VOCAB_SIZE=151936）
    - HuggingFace Tokenizer集成
    - embedding权重共享（untie_embeddings_and_output_weights）
    - HF↔Megatron权重转换脚本调试
  - 训练排查：处理了4B模型64卡训练的显存不足、梯度爆炸、checkpoint加载等问题，形
  成了完整的调试文档。

  4. 训练实验

  - 基座模型：Qwen3-1.7B
  - 训练规模：64 GPU（H200/H20），10B tokens
  - 关键超参数：
    - 学习率：基座的1/10，使用WSD（Warmup-Stable-Decay）调度
    - Global Batch Size / Micro Batch Size根据GPU型号调整
    - 使用BF16混合精度训练
  - Micro-annealing验证：在训练过程中周期性切换为纯高质量数据+cosine衰减的短期退
  火，以更准确评估当前checkpoint的真实能力。
  - H200 vs H20对比：H200吞吐量约为H20的4倍，显存48GB vs 96GB但计算效率更高。   

  5. 评测体系建设

  - 私域评测体系（一期）：构建了面向高德业务场景的评测框架，覆盖意图理解、地理知
  识问答、POI推荐等任务。
  - 后训练LLM评测体系（V0）：设计了通用能力+垂域能力的多维度评测方案。
  - 意图理解评测：针对新框架设计了意图识别的专项评测集和评估指标。
  - 评测方法：结合自动化评分（准确率、F1等）和人工评估（相关性、流畅性等）。
